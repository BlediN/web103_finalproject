# User Stories

Reference the Writing User Stories final project guide in the course portal for more information about how to complete each of the sections below.

## Outline User Roles

- **Guest**  
Can browse layoff stories, view statistics, and access content without creating an account.

- **Authenticated User**  
Can submit, edit, and delete their own layoff stories, choose to post anonymously, and manage their profile.

- **Admin**  
Has full access to moderate content, remove inappropriate posts, reset the database, and ensure overall data quality.

## Draft User Stories

1. As a **guest**, I want to browse layoff stories on the feed and by company so that I can learn from other people’s experiences and understand trends at specific organizations without needing an account.

2. As an **authenticated user**, I want to submit my own layoff story so that I can share details about my severance, timeline, and experience with others.

3. As an **authenticated user**, I want to edit or delete one of my posted layoff stories so that I can keep my information accurate and up to date.

4. As a **guest**, I want to open a layoff story in a detail panel or modal so that I can read the full experience without leaving the main feed.

5. As a **guest**, I want to filter layoff stories by company, industry, job type, or year so that I can quickly find stories that are most relevant to me.

6. As a **guest**, I want to sort layoff stories by severance length or newest post so that I can compare the most recent or most informative entries more easily.

7. As an **authenticated user**, I want the app to validate my layoff entry before it is submitted so that incomplete or unrealistic data is not saved.

8. As a **guest**, I want to view statistics such as average severance and average job search length by industry so that I can better understand broader layoff trends.

9. As an **authenticated user**, I want to view and manage the layoff stories I have posted from my profile page so that I can keep track of my contributions.

10. As an **authenticated user**, I want to post my story anonymously so that I can share sensitive information without revealing my identity.

11. As an **admin**, I want to remove inappropriate or inaccurate layoff stories so that the platform maintains trustworthy and high-quality data.
